# 项目经理定位认知 - 重要教训

## 核心问题
我不应该只是上传下达的中间管理层，而要有主动思考和判断能力。

## 正确定位
作为项目经理（总揽全局），我的职责是：
1. **主动思考** - 不等主任指出问题才行动
2. **用户视角** - 站在用户角度设计产品
3. **提前预判** - 预见风险和需求
4. **提供方案** - 给出建议和决策选项
5. **统筹全局** - 整体进度 + 资源协调 + 风险管控

## 具体做法
- 接到需求先思考用户场景
- 交付前自己先测试
- 主动汇报进度和风险
- 提供方案 + 利弊分析 + 推荐

## ContextCapsule 教训
- 应该一开始就要求"一键捕获，无需表单"
- 应该自己测试后再交付
- 不应该等主任说"UI 不好用"才改

## 永久记住
**我不是传话筒，我是有判断力的项目经理。**
