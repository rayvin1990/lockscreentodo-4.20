## 技术决策记录

### 决策 1: 选择文件系统而非数据库

**日期:** 2026-03-10  
**决策:** 使用 Markdown + JSON 文件存储，不使用 SQLite/MySQL

**理由:**
- 简单易懂，用户可直接编辑
- 易于备份和版本控制
- 无外部依赖
- 适合个人使用场景

**影响:**
- ✅ 降低复杂度
- ✅ 便于迁移
- ⚠️ 大规模数据性能可能受限（但个人使用足够）

### 决策 2: 命令行优先

**日期:** 2026-03-10  
**决策:** 先开发 CLI，GUI 后续考虑

**理由:**
- 开发速度快
- 易于集成到现有工作流
- OpenClaw 本身是命令行工具
- 可远程使用

**影响:**
- ✅ 快速交付
- ✅ 低资源消耗
- ⚠️ 学习成本略高（但文档可弥补）

### 决策 3: Python 标准库优先

**日期:** 2026-03-10  
**决策:** 尽量使用标准库，减少外部依赖

**理由:**
- 降低安装复杂度
- 提高兼容性
- 减少维护成本

**影响:**
- ✅ 开箱即用
- ✅ 稳定性高
- ⚠️ 部分功能实现稍复杂
