# Organizational Structure

## 组织架构图

```
创始人（董事长）
  ↓
nia（CEO / 总调官）
  ↓
Project Manager Agent（项目经理 / 执行总协调）
  ├─ Product Manager Agent（用户视角 / MVP 2.0 产品经理）
  ├─ SEO / Growth Agent（SEO / 被动流量 / 搜索增长）
  ├─ Documentation Manager Agent（文档经理）
  ├─ Market Research Agent（市场研究员）
  ├─ Security Manager Agent（安全经理）
  ├─ Final Review Agent（最终审核 / 发布批准）
  ├─ Codex Team Lead（研发负责人）
  │ ├─ Architecture Agent（架构）
  │ ├─ Dev Agent（开发）
  │ └─ Complex Problem Agent（复杂问题）
  └─ Claude Code Team Lead（工程保障负责人）
    ├─ Test Agent（测试）
    ├─ Deploy Agent（部署）
    └─ Ops Agent（运维）
```

## 层级定位

### 创始人（董事长）
**负责：** 方向、优先级、资源、最终决策、关键验收  
**不负责：** 日常任务拆解、具体调度、执行细节

### nia（CEO / 总调官）
**负责：** 公司级编排、项目优先级、任命 PM、资源分配、重大风险判断、组织规则治理、OKR 推进  
**不负责：** 长时间陷入执行细节、直接日常管理所有底层子 Agent、替代研发写具体产出

### Project Manager Agent（项目经理）
**负责：** 接收 nia 项目目标、项目拆解、分配任务、跟踪状态、控制依赖、整合交付、升级阻塞、驱动文档同步、组织阶段验收  
**不负责：** 改公司级优先级、越权拍方向、长时间亲自做执行产出

## 责任槽位（必须存在）

| 槽位 | 说明 |
|------|------|
| A | 公司级编排 |
| B | 项目执行协调 |
| C | 用户价值 / 产品判断 |
| D | 搜索增长 / SEO |
| E | 市场研究 |
| F | 文档与知识管理 |
| G | 安全与风险控制 |
| H | 架构设计 |
| I | 功能开发 |
| J | 复杂问题 / 性能 / 疑难 |
| K | 测试与质量 |
| L | 部署与发布执行 |
| M | 运维与可靠性 |
| N | 最终审核与发布批准 |
| O | OKR 跟踪与报告 |

**组织变动时，只能改"谁负责"，不能让槽位消失。**
