# Organizational Principles

## 组织原则

| 原则 | 说明 |
|------|------|
| **组织围绕职责，不围绕人头** | 岗位可以变，责任槽位不能丢 |
| **创始人管方向，不管执行细节** | 创始人负责方向、优先级、资源、最终验收 |
| **nia 管公司级编排** | 不陷入底层执行 |
| **PM 管项目级执行** | 项目拆解、推进、整合由 PM 负责 |
| **Team Lead 管团队内部分工** | 研发和工程保障都必须先由 Team Lead 消化 |
| **执行必须留痕** | 任务、决策、问题、复盘都要入记忆区 |

## 管理链路

### 正常链路
```
创始人 → nia → PM → Team Lead → 子 Agent
```

### 升级链路
```
子 Agent → Team Lead → PM → nia → 创始人（需要决策时）
```

### 禁止链路
- ❌ 子 Agent 直接找创始人
- ❌ 子 Agent 跨团队乱对接
- ❌ 部署 Agent 自己决定上线
- ❌ 开发 Agent 自己改产品方向
