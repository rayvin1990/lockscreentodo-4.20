# 市场战略 - 美国优先

## 核心定位

**只做海外市场，首选美国，不做国内产品。**

这是团队的**核心战略**，所有产品必须遵循。

---

## 为什么选择美国市场

| 因素 | 美国市场 | 国内市场 |
|------|----------|----------|
| 付费意愿 | 强（$5-20/月接受度高） | 弱（习惯免费） |
| 汇率优势 | 美元收入（1 USD = 7.2 CNY） | 人民币 |
| 竞争环境 | 相对公平 | 内卷严重 |
| 合规风险 | 低（标准明确） | 高（政策多变） |
| 用户规模 | 全球市场 | 仅国内 |

---

## 产品要求

### 1. 语言
- **默认 UI：** 英文
- **文案风格：** 简洁直接，符合欧美习惯
- **不做：** 中文 UI（除非用户主动切换）

### 2. 定价
- **货币：** USD（美元）
- **价格区间：** $5-20/月 或 $49-199 一次性
- **支付：** Stripe / PayPal / Gumroad

### 3. 合规
- **隐私政策：** GDPR/CCPA 合规
- **数据存储：** 本地优先（localStorage/IndexedDB）
- **追踪：** 最小化（不收集个人数据）

### 4. 部署
- **托管：** Vercel / Netlify（全球 CDN）
- **域名：** .com（不用 .cn）
- **速度：** 全球访问优化

### 5. 营销
- **渠道：** Reddit / Twitter / Product Hunt
- **内容：** 英文博客/教程
- **不做：** 微信/微博/小红书

---

## 文档规范

所有 PRD 和原型**必须**在开头注明：

```markdown
## 🌍 市场定位

**目标市场：** 美国/海外  
**首选用户：** 美国 [目标用户群体]  
**不做：** 中国市场  
**语言：** 英文 UI  
**货币：** USD  
```

---

## 团队原则

1. **产品设计阶段** 就明确海外定位
2. **不用国内思维** 做产品（如裂变、拉新）
3. **专注价值** 而非流量
4. **美元收入** 而非人民币
