# MVP 2.0 Strategy

## MVP 2.0 定义

**MVP 2.0 = 小 + 可用 + 差异化 + 值得传播**

## 规则

| 规则 | 说明 |
|------|------|
| **核心功能 ≤ 3** | 超过 3 个核心功能的不是 MVP 2.0 |
| **用户价值 10 秒内能说清** | 用户第一眼必须知道价值 |
| **差异化至少来自 2 项** | AI 杠杆更强、用户体验更好、自动化更强、更垂直、任务完成更快 |
| **没有差异化，暂停开发** | 退回 Product Manager 重新定义 |
| **没有清晰尖叫点，退回** | 必须有值得传播的亮点 |
| **没有验证计划，不进入研发** | 必须有明确的验证方式 |

## Product Manager 判断清单

每个 MVP 必须回答：

| 问题 | 检查点 |
|------|--------|
| 核心功能是否超过 3 个？ | ≤3 |
| 用户第一眼是否知道价值？ | 10 秒内理解 |
| 是否有差异化？ | 至少 2 项来源 |
| 是否有一个值得传播的亮点？ | 尖叫点 |
| 首次体验是否顺？ | 无阻碍 |
| 用户完成关键任务是否更快？ | 比竞品快 |
| 这东西是不是"普通但能用"还是"值得推荐"？ | 值得推荐 |

## 差异化来源（至少 2 项）

| 来源 | 说明 |
|------|------|
| **AI 杠杆更强** | 用 AI 实现竞品做不到的自动化 |
| **用户体验更好** | 更简洁、更直观、更快速 |
| **自动化更强** | 用户操作更少，系统自动完成更多 |
| **更垂直** | 专注特定用户群体或场景 |
| **任务完成更快** | 用户完成核心任务的时间更短 |
